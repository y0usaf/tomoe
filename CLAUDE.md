# tomoe

Wayland compositor (Smithay + embedded Lua), Rust workspace, built via Nix
flake (see doctrine: build/verify through `nix build` / `nix flake check`,
not bare cargo).

## Debug launches MUST be time-boxed

Launching tomoe (or `tomoe-session`) with the tty backend takes over DRM
master — the user's screen goes black and they cannot tell if the session
is alive. NEVER launch it unbounded. Always wrap it and log to a file:

    timeout 5 tomoe-session > /tmp/tomoe-debug.log 2>&1; echo "exit: $?"

Exit 124 means the timeout killed it (expected). Tell the user before
launching that the screen will blank for ~5 seconds and return on its own.

## Architecture map — do not re-explore

`ARCHITECTURE.md` is a **generated** map of the workspace: the crate
dependency graph plus per-crate module/type/trait trees. It is regenerated by
`scripts/gen-arch.sh` and its freshness is enforced by `nix flake check`
(`checks.arch-fresh`), so it is always accurate — read it instead of spawning
exploration agents to map the codebase. For rationale, dataflow, and
invariants (which the generated map cannot capture), read `DESIGN.md`.
After adding/removing crates, modules, or public items, rerun
`scripts/gen-arch.sh` (jq and cargo-modules are in the dev shell; run inside
`nix develop` so build scripts find native libs) and commit the result, or
`arch-fresh` will fail.
